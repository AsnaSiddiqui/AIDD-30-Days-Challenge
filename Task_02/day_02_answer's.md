## 2. Vibe Coding vs Specification-Driven Development

### A. Why does Vibe Coding usually create problems after one week?

Vibe Coding me koi planning, specs ya clear direction nahi hoti.
Start me sab easy lagta hai, lekin aik haftay baad code messy, inconsistent aur change karna mushkil ho jata hai, kyunki shuru se koi structure follow hi nahi hota. Har naya feature add karte waqt confusion create hoti rehti hai.

### B. How would Specification-Driven Development prevent those problems?

SDD me pehle clear specs likhi jati hain, yani kya banana hai, kis tarah chalega, aur kis rule ke sath.
Jab implementation specs ke basis par hoti hai to code clean, predictable aur maintainable hota hai.
Aise me future changes bhi aasani se ho jate hain, aur vibe coding wali messy situation bhi nhi hoti.



## 3. Architecture Thinking

### A. How does architecture-first thinking change the role of a developer in AIDD?

Architecture-first approach developer ko sirf code writer se system designer bana deti hai.
Developer pehle pura structure, flows aur components ka plan banata hai, phir AI uske according code generate karta hai.
Is se developer ka role low-level coding se upar uth kar high-level reasoning aur design par aa jata hai.

### B. Explain why developers must think in layers and systems instead of raw code?

Layers me sochne se system clean, organized aur scalable ban jata hai.
Agar developer sirf raw code me sochay to project jaldi bigar jata hai, kyunki sab cheeze mix ho jati hain.
Layered thinking se har part ka role clear hota hai models, logic, UI, agents. sab alag rehte hain aur system maintain karna bohot easy ho jata hai.
