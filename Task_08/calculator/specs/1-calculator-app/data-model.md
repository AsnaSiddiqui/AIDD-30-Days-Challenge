# Data Model: Calculator Web Application

**Date**: 2025-12-01

## Entities

### 1. Arithmetic Expression
-   **Description**: A string representing a mathematical calculation provided by the user.
-   **Attributes**:
    -   `expression_string`: String (e.g., "5 + 3", "10 / 2.5", "10 / 0").
-   **Validation Rules**:
    -   Must contain only digits, decimal points, and allowed operators (+, -, *, /).
    -   Must not be empty.
    -   Must form a syntactically valid arithmetic expression.

### 2. Result
-   **Description**: A numerical value derived from evaluating the `Arithmetic Expression`.
-   **Attributes**:
    -   `value`: Float or Integer (e.g., 8, 5.5, 42).
-   **Validation Rules**: N/A (Result is derived, not user input).

### 3. Error Message
-   **Description**: A string displayed to the user when an `Arithmetic Expression` is invalid or results in an error (e.g., division by zero).
-   **Attributes**:
    -   `message_string`: String (e.g., "Division by zero is not allowed", "Invalid expression syntax", "Invalid input").
-   **Validation Rules**: N/A (Error message is generated by the system).
